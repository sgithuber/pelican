working with python virtual environment in bash

Thu 21 January 2021

as python3 right now is 3.9.1, create virtual environment is something like:

[root@localhost ~]# python3 --version
Python 3.9.1

[root@localhost ~]# python3 -m venv ./
...

this means the folder name will be the (folder name) at the begining.

thus we can do

[root@localhost ~]# vim /root/.bashrc
...
alias activate='deactivate 2> /dev/null; source ./bin/activate'  
:x

there're two good reasons:

  • simple type active, other than source ./bin/activate.
  • project A virtual environment is activated, we can directly type activate to activate project B virtual environment without tying deactivate, easy and time saving.
